（施工中）【从零到一】搭建一个完整项目的解决方案

发表于 2018-04-21 | 分类于 技术

文档施工中…

0、概述

事情源于陈新磊同学的一次提问。

4.19那天下班前，陈新磊过来找我，咨询了一下关于docker部署中的问题，同时也问了我一下关于SSL证书（HTTPS）的注册和部署相关的问题。我就简单的给他解释和演示了一下工作流程。

但是后来仔细一想，发现这个事情倒是非常适合拿出来跟大家做技术分享的。虽然之前的每一个步骤都零零星星的跟大家解释过，包括Ubuntu、Docker、还有注册域名、云主机等。却没有把所有细节都连在一起串讲一次。

那么这次，就趁着这个机会，把这整个项目流串在一起，从零到一，给大家介绍一下，搭建一个完整项目的解决方案。

1、目标

我们这次需要达到的目标其实也很简单，注册一个自己的域名，购买一个云主机，申请免费1年的SSL证书，并且在云主机中运行docker，配置属于自己的开发/测试环境，搭建一个属于自己的网站。在最后的番外篇中，会简单描述一下如何搭建一个完全免费的个人博客，并且绑定属于自己的域名。

2、流程

2.1、步骤

  • 注册域名 + 实名认证
  • 注册云主机 + 备案
  • 为域名申请免费SSL证书
  • [非必选]优化Ubuntu基础设计
  • 安装Nginx
  • 安装Docker
  • 安装Node.js （or Docker安装Node.js) (or 打包整合Node.js绿色包）
  • Docker 安装 MySQL
  • Docker 安装 Docker Registry
  • 配置SSL证书
  • Docker 打包node.js项目image，并上传Docker Registry
  • Docker 下载自建image并运行

2.2、注册域名 + 实名认证

因为习惯原因，我的域名是在「阿里云」注册的。因为阿里云最早提供的手机APP，可以直接在手机上对云服务的各种管理非常方便。注册过程可以参考阿里的「官方文档」。

2.3、注册云主机 + 备案

2018年上半年，腾讯云搞了个学生主机1折优惠活动，1C2G的主机，主要10块钱一个月，最高可以续满3年，一共360块钱（甚至早期还有活动bug，可以申请降配置为1C1G，差价补成时长，即360块6年的云主机）。

所以我就在「腾讯云」注册了一台云主机。然后为了方便开通80端口，也为了可以认证使用HTTPS，就去对云主机申请了备案（其实是一开始不知道HTTPS的443端口不需要备案，也不知道是不是个bug）。

备案流程并不复杂： 腾讯云的「备案流程图解」

  • 首先是认证主体信息，其中拍照包括身份证正反面（武汉地区还需要「武汉市居住证」正反面），还有手持身份证的半身照。

  • 然后就是生成一张备案资料表，其中还有包括备案用的域名、网站名称等手填资料，

  • 第一轮拍照上传完之后，腾讯云的工作人员会对资料进行一个初审，一般在一个工作日内，早上提交下午就能收到反馈。初审完成后，就会要求补充一个带背景（幕布）的手持身份证照。

  • 第二轮中要求的幕布手持照片，有两种种解决方案：

    1. 是到指定的拍照地点拍照，由工作人员审核完资料后，再给上传照片。
    2. 是填写一个收货地址，腾讯免费给你邮寄一份拍照幕布，自己拍了再行上传。
  • 上传完第二轮的幕布照片后，大概也需要1个工作日，腾讯云的工作人员审核过后，就会给你电话联系，确认照片是否是本人，是否有修改编辑过等等。
  • 电话联系过后，腾讯云的工作人员就会将你的备案资料提交至当地管局进行最终审核。
  • 最后等待大约1个自然月左右的时间（20个工作日），就会收到备案通过的手机短信和电子邮件。然后大概过半天到一天左右（并不是马上），80端口就可用了。

2.4、为域名申请免费SSL证书

注册SSL证书我们用到的是「腾讯云」。

为什么用腾讯云呢？嗯，最早的历史原因，是因为腾讯云应该是最早提供「免费SSL证书」的国内（规模够大够靠谱的）服务商，国外的因为大部分都是英文的，而且使用起来比较麻烦，所以干脆直接在腾讯云搞。

在此之后，「阿里云」也跟着放出了申请免费SSL证书的服务了。但是由于入口比较隐秘，并且免费选项也比较隐秘，所以不做推荐。但我这里还是会在小节底部补充一下阿里云申请免费证书的方式，详情参见「阿里云申请免费CA证书方式」。

腾讯云申请SSL证书的流程就相对简单了，毕竟放的比较显眼，没那么隐秘。

2.5、[非必选]优化Ubuntu软件源

由于某些不可抗拒原因，国内连接国外的各种软件源会非常非常的卡，各种连不上。国内的几大巨头，为了解决这种问题，专门搭建了自己的软件源，方便了国内不擅长用「梯子」的程序员们的开发。

国内的云主机服务商，自带的Ubuntu操作系统，一般都是做过一定程度的定制，如软件源之类的都默认指向了服务商自己的源，省去了我们改源的操作。

2.6、安装Nginx

其实安装Nginx有两种方式，一种直接在系统中安装，另一种是通过docker来安装。

我使用的是通过系统自带的apt-get模式来进行安装的。目前从短期使用下来看，并没有本质上的区别。从长远来看，有利有弊。弊端是升级可能不如docker版的方便，如果卸载估计也没docker版的直接rm那么彻底。但是好处是直接调用系统底层，刨除性能问题外，还有方便调试和修改等优势。

那么，就基于我现在正在使用的直接安装的方式来讲解一下安装流程。

  • 首先是更新一下安装源
  • 接着安装nginx。

安装完成后，通过IP或绑定的域名打开，应该就可以看到nginx的欢迎页面了。（刚申请的主机不会马上被Ban80端口，还有个测试的机会。）

其中有一个需要比较关注的点：FROM node:6-alpine

该段代码在制定用的node版本为6之余，-alpine（阿尔卑斯/阿尔派）作为参数大大的缩减了镜像的空间。（从666M -> 62M）

尺寸缩小了近10倍。为push镜像节约流量打下了很重要的基础。

对于选用版本的参考建议，可以参考官方说明：「Docker - node」
