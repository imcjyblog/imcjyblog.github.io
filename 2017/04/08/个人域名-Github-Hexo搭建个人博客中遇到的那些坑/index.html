Github上的设置

  • 新建的repository必须是username.github.io，如我的imcjyblog.github.io。

  • 然后还需要在项目的下方新建一个CNAME文件，无后缀名。

  • 里面就一句话，写上你的域名。（无需 www ）

  • 最后，记得同时放这个CNAME文件到你的本机的Hexo博客的 \source 文件夹下面。

这样才能确保你的 hexo d 的推送能保证这个CNAME可以一直存在。

域名的设置

在阿里云上的域名的设置其实也是一个大坑，我也是反反复复试错了十几回才最终搞定，现在直接把经验分享出来，省的再来踩坑的。

阿里云上CNAME的设置

直接上关键设置图：

192.30.252.153

192.30.252.154

这两个IP地址是github的官方建议地址，主要是通过A绑定之后，在github上的那个CNAME会让github来反向解析我们的域名，从而找到我们的github pages。

然后，阿里云上的CNAME，直接www（你非要用blog我也不拦你）来绑定你的本身的github pages的二级域名，如我的imcjyblog.github.io.。注意，这里用的是.io，并且，最后那有一个.结尾的，不要漏了。

在以上配置都没错之后，你的域名应该就可以直接访问你的github pages了…

最后，祝大家建博愉快…
